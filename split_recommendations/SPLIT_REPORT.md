### 拆分原则

1. **单一职责** - 每个模块只负责一个功能域
2. **依赖倒置** - 依赖抽象不依赖具体实现
3. **接口隔离** - 使用 Protocol 定义接口
4. **向后兼容** - 通过 __init__.py 导出保持兼容

### 标准拆分结构

```

├── __init__.py      # 导出公共 API
```
